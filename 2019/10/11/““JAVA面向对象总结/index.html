““JAVA面向对象总结

第二章 类和对象

* 1.面向对象概念：

​ 面向对象就是把一个项目，一个事情分成更小的项目，或者说分成一个个更小的部分，每一部分负责什么方面的功能，最后再由这些部分组合而成为一个整体。

* 2.类的创建：

语法：public class 类名{}

步骤：File—>new—>class—>类名

* 3.创建对象：

语法：类名 对象名=new 类名();

​ 对象名.方法名(); 对象名.属性

* 4.构造方法：

方法名和类名相同,没有返回值

* 5.构造方法重载:

在同一个类中，方法名相同，参数个数，顺序，类型不相同的构造方法

* 6.构造方法的执行：

new 类名();—>执行构造方法

super();—>执行了父类的构造方法

* 7.static静态修饰符:

静态修饰符可以修饰属性、方法、代码块，静态方法中不能使用this和super关键字。

如何使用：

类名.属性 类名.方法名

* 8.java内存（栈和堆，static，方法区，gc（垃圾回收机制））

第三章 面向对象三大特征—封装

* 1.封装的概念：

​ 封装指的是将对象的状态信息隐藏在对象内部，不允许外部程序直接访问对象内部信息，而是通过该类所提供的方法来实现对内部信息的操作访问。

* 2.封装的步骤：

​ 1.把属性私有化（private）

​ 2.对这些属性设置get和set方法

​ 3.调用get和set方法

* 3.封装的作用：

​ 避免一些用一些不合乎常理的操作

* 4.修饰符：

public--->公共的：在一个项目中都可以访问，权限最高

​ private—>私有的：只能在本类中进行访问，权限最低

​ protected—>安全的，受保护的：同一个包能进行访问，继承中之类可以访问父类

​ default—>默认的：同一个包能访问

* 5.好处：

​ 提高了安全性 提高了复用性 提高了实现细节

第四章 面向对象三大特征—继承

* 1.继承的概念：父亲儿子，电影《西虹市首富》

​ 比如FeiGe快递系统中有三个实体类Person（人），Courier（快递员）,Customer（客户）类，Courier,Customer都属于（派生类）Person，所以都继承（extends）自Person类。

* 2.好处：

​ 提高了代码的重用性 提高了代码的扩展性

* 3.继承的关键字：extends

* 4.继承的语法：

​ public class 类名 extends 父类{}

* 5. 如何使用：

在子类中，可以直接调用父类的非私有属性和方法，可以用super.方法 super.属性。

* 6.父类构造方法的调用：super();

调用父类的构造方法必须在子类中调用，必须实在之类构造方法的第一行代码

* 7.重写@Override：必须继承

​ 方法名一致，参数个数，顺序，类型全部一致
