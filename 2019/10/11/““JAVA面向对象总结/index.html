““JAVA面向对象总结
发布于 2019-10-11   更新于 2019-10-14

第二章 类和对象

* 1.面向对象概念：

​ 面向对象就是把一个项目，一个事情分成更小的项目，或者说分成一个个更小的部分，每一部分负责什么方面的功能，最后再由这些部分组合而成为一个整体。

* 2.类的创建：

语法：public class 类名{}

步骤：File—>new—>class—>类名

* 3.创建对象：

语法：类名 对象名=new 类名();

​ 对象名.方法名(); 对象名.属性

* 4.构造方法：

方法名和类名相同,没有返回值

* 5.构造方法重载:

在同一个类中，方法名相同，参数个数，顺序，类型不相同的构造方法

* 6.构造方法的执行：

new 类名();—>执行构造方法

super();—>执行了父类的构造方法

* 7.static静态修饰符:

静态修饰符可以修饰属性、方法、代码块，静态方法中不能使用this和super关键字。

如何使用：

类名.属性 类名.方法名

* 8.java内存（栈和堆，static，方法区，gc（垃圾回收机制））

第三章 面向对象三大特征—封装

* 1.封装的概念：

​ 封装指的是将对象的状态信息隐藏在对象内部，不允许外部程序直接访问对象内部信息，而是通过该类所提供的方法来实现对内部信息的操作访问。

* 2.封装的步骤：

​ 1.把属性私有化（private）

​ 2.对这些属性设置get和set方法

​ 3.调用get和set方法

* 3.封装的作用：

​ 避免一些用一些不合乎常理的操作

* 4.修饰符：

public--->公共的：在一个项目中都可以访问，权限最高

​ private—>私有的：只能在本类中进行访问，权限最低

​ protected—>安全的，受保护的：同一个包能进行访问，继承中之类可以访问父类

​ default—>默认的：同一个包能访问

* 5.好处：

​ 提高了安全性 提高了复用性 提高了实现细节

第四章 面向对象三大特征—继承

* 1.继承的概念：父亲儿子，电影《西虹市首富》

​ 比如FeiGe快递系统中有三个实体类Person（人），Courier（快递员）,Customer（客户）类，Courier,Customer都属于（派生类）Person，所以都继承（extends）自Person类。

* 2.好处：

​ 提高了代码的重用性 提高了代码的扩展性

* 3.继承的关键字：extends

* 4.继承的语法：

​ public class 类名 extends 父类{}

* 5. 如何使用：

在子类中，可以直接调用父类的非私有属性和方法，可以用super.方法 super.属性。

* 6.父类构造方法的调用：super();

调用父类的构造方法必须在子类中调用，必须实在之类构造方法的第一行代码

* 7.重写@Override：必须继承

​ 方法名一致，参数个数，顺序，类型全部一致

第五章 面向对象三大特征—多态

* 1.多态的概念：

​ 指允许不同类的对象对同一消息做出响应即同同一消息可以根据发送对象的不同而采用多种不同的行为方式。

* 2.好处：

​ 可替换性 可扩充性 接口性 灵活性 简化性

3.多态的必要条件：

​ 继承

​ 重写

​ 父类引用指向子类对象（向上转型）

* 4.多态的使用：

​ 向上转型：<父类型><引用变量名>=new<子类型>（）；

向下转型：<子类型><引用变量名>=（<子类型>）<父类型的引用变量>；

第六章 抽象

* 1.抽象的概念：

​ 如果一个类没有足够的信息去描述一个具体的对象那我们就称之为抽象类。

* 2.抽象的关键字：

abstract 可以修饰类，可以修饰方法

* 3.抽象的用法：

* 4.final修饰符：最终的

​ 修饰变量，修饰方法，修饰类

​ 1.当修饰变量时，变量转换成常量，常量的值不能改变

​ final int a=18;//a=18;不能改变，必须给一个初始值。

​ 2.当修饰方法时，该方法不能被重写

​ 3.当修饰类时，该类不能被继承

第七章 接口–interface

* 1.接口的概念：

​ 在软件中接口是一种规范和标准他们可以约束类的行为是一些方法特征的集合但是没有方法的实现。

* 2.接口的关键字——interface

* 3.接口的语法：

  • File—>new—>interface//创建接口

* 4.接口的用法：

* 实现接口的关键字 implements

注意接口可以进行多继承

  • public interface 接口名 extends 接口名，接口名{}

  • 一般用法

  • public class 类名 extends 父类 implements 接口，接口{}

    第八章 集合框架

    * 1.ArrayList

    1.用法

    • ArrayList list=new ArrayList();
    • list.add();//添加数据
    • list.get();//获取数据

* 2.Map

用法

  • Map map=new HasMap();
  • map.put(key,value);//key不能重复
  • map.get(key);//获取key对象的value

* 3.泛型

​ 数组和集合的比较

数组 集合
长度 长度是固定的 长度是可变的
存储对象 可以存储基本数据类型，也可以存储对象的引用 只能用于存储对象的引用
对象类型 必须是相同类型的数据 对象可以是不同类型的

第九章 文件操作

* 1.file类对象的主要作用是用来获取文本本身的一些信息

* 2.流

​ 流具有明确的方向性，当向一个流写入数据时，这个流被称为输出流，输出流可以将信息送往程序的外部。

第十章 多线程

* 1. 线程的概念

线程可以理解成是在程序中独立运行的子任务

* 2.线程的创建

* 创建线程的两种方式：

1.实现Runnable接口和继承Thread类。

步骤：

（1）定义一个类实现Runnable接口

（2）实现接口中唯一的run()方法，该方法是线程开启时要执行的歌曲下载

（3）在代码中声明一个线程实体Thread myThread=new Thread(download);//download为实现Runable 接口的类的实体对象

（4）调用myThread.start()方法启动新创建的线程。

2.继承Thead创建线程

步骤：

（1）创建类MyThread并继承Thread类

（2）实现类MyThread的run()方法，该方法是线程开启时要执行的下载

（3）生成MyThread类的对象，MyThread myThread=new MyThread();

（4）调用myThread.start()方法启动新创建的线程。

3.多线程的同步是“排队”并非同时进行

4.线程死锁

当synchronized关键字修饰一个方法的时候，该方法叫做同步方法

当synchronized方法执行完或发生异常时，会自动释放死锁 。
