web前端总结（三）

七.关于DIV+CSS布局

​ 内联元素修饰，外联元素布局。如果要修饰内联元素标签，就要使用b标签或者u标签，修饰不换行的内容，就要用内联标签，但内联标签不干净，自带有修饰效果，所以我们需要一个干净的没有任何自带修饰效果的内联元素标签，w3c组织提出干净的内联标签这个标签是span标签。
​ 与之相对象和匹配的，提出了一个干净的会自动换行的外联元素的标签，叫div标签。
​ span内联+div的外联标签都是干净的标签，div专门用于布局，span专门用于修饰。
​ 因为网页中的每一块内容都是独立存在的，每一块都是布局，所以针对id选择符的特点，我们在div标签中如果要修饰div的css属性，用id选择符。修饰内联的span用class选择符。因为类可以有多个，又专门进行样式修饰使用的，所以span中使用class选择符。

*1.内联元素的属性：
  • 字体font:
  • 文本text:
  • 锚伪类，控制鼠标经过，鼠标选中，鼠标访问过后的效果

在网页使用锚伪类时，顺序不能改变，分别就是鼠标经过，鼠标选定，鼠标点击。

*2.修饰外联元素的css属性：

外联元素是一块，所以也叫块级元素，所以就有了网页中的盒子模型。

盒子模型是指网页中的内容，盒子模型–border边框、content内容、内间距padding、margin外间距。所有的外联元素和内联元素都可以理解为一个盒子的存在。盒子模型中有宽度、高度、边框线、内间距、外间距

  • width宽度，宽度有固定宽度和自适应宽度auto两种。

inherit(继承)宽度，当父级的盒子指定固定宽度，而包含的盒子不指定宽度，包含的盒子会继承父级盒子的宽度。由此可见，宽度可以继承，高度可以被内容撑起。
注意：只有外联元素才有宽（设定宽度网页效果会生效）。内联元素也有宽高，但是是随着内容而变化的宽高，也就是指定内联元素的宽高是不会有效果的。

  • height高度，高度有固定高和内容撑起的高两种。

注意：宽度的特征是会继承，高度的特征是不会继承父级元素的高，但会被内容撑起来。
内容可能超过父级元素标签的宽高，所以我们用overflow控制内容溢出的属性。
overflow的属性一般在父级元素标签中使用：
overflow: hidden;当子级的标签超过父级标签，超出的部分隐藏掉。
overflow: auto;超出部分会显示滚动条，如果不超出，就不显示滚动条。
overflow: scroll;无论是否超出，都强制显示滚动条。
总结：当使用宽度和高度的属性时，尽量写父级元素大盒子的宽度，别写高度，里面的小盒子只写高度，别写宽度。
当宽度高度都是固定的高度，要使用overflow的属性。

  • 边框线border，可以同时定义四条线，可以定义单独一条线，一般我们采用简写的方式：

注意：线的类型dashed虚线 dotted点线 solid实线 double双线（线的粗细不能低于1px）

  • padding内间距，表示边框到内容之间的距离。设定盒子的内间距，会撑大盒子。

    内间距可以定义四个方向：当按照顺时针的方式进行每个padding的设定，盒子会被撑大相应的尺寸，和内容的间距，是padding-top和padding-left是真实的间距。当定义了padding-top间距，padding-bottom值失效（虽然值失效，这里失效是内容和边框间距失效，但撑大盒子的值并未失效），依次类推，定义padding-left，padding-right的值失效，但撑大效果并未失效。

固定宽高的盒子尺寸=宽度+内间距+border线
总结：调整内容在大盒子中的位置，可以采用内间距padding，但内间距会撑大盒子，所以一定有属性也可以改变内容的间距，且不撑大盒子。这就是内容的外间距。
继承大盒子的宽度的小盒子设定内间距为什么不受内间距撑大盒子的影响？
当继承的宽度（不是固定设定的宽度）设定内间距不会撑大盒子。

  • 外间距margin，表示内容到边框的间距。简写的方式和内间距一样，也是顺时针进行设定。

注意：外联元素和内联元素都可以使用内间距和外间距，但生效的位置不一样。
外联元素进行内间距和外间距生效的是四个方向。
内联元素的内间距生效的距离是左右和下，外间距生效的范围是水平的左右。
利用外间距margin：0 auto；表示一个盒子的居中，但要注意，没有固定宽度的盒子是无法居中的。
通常的盒子模型指的是内联元素还是外联元素？
盒子模型–border边框，content内容，内间距padding，外间距margin
内联元素也可以理解为盒子模型，因为都有边框内容和间距，但要注意，内联元素宽和高都是依靠内容撑起的，内间距依然撑大内联元素的盒子，外间距可以控制内联元素的外间距，但生效方式只有水平横向的位置。

  • 浮动float–盒子脱离了body文档流进行水平方向的移动。

清除浮动clear：清除浮动并不是设定盒子不再浮动回到body层，而是清除前面盒子布局对于后面盒子布局的影响，可以操作浮动层的内容换行。

  • 定位position–盒子脱离body的文档流，进行二维坐标的定位。二维坐标就是top上，下bottom，左left，右right，当使用定位时，上下二维坐标取一个值，左右二维坐标取一个值，就能定位到一个具体的位置。

***相对定位relative：相对盒子当前的位置进行移动，原先位置内容消失，脱离文档流。

***绝对定位absolute：相对body的边框进行移动，原先位置内容取消，脱离文档流。

注意：当大盒子定义了相对定位，小盒子定义绝对定位，小盒子的绝对定位不再参照body，而是参照大盒子进行移动。
固定定位fixed：始终参照body的位置进行定位。
使用定位时，如果定位的盒子发生了重叠，使用z-index来表示盒子层叠的优先级。

  • visibility元素是否可见: hidden隐藏;visibility: hidden;表示盒子被隐藏但会有占位的效果。
  • display转换：实现内联元素和外联元素的相互转换。
    display: inline;外联转内联
    display: block；内联转外联
    display：inline-block；行内块级元素，在一行显示，但可以定义宽高。
